# 発表メモ：ピピットセルフ連携（頭に入れる用）

## 位置づけ（1行で言うと）
鳩ナビは、ピピットセルフを置き換えるものではなく、将来的にピピットセルフの買い物体験と接続可能な親子向け体験プロトタイプです。現時点では独立したWebデモとして動作し、将来的にはピピットセルフアプリ内統合、店舗貸出端末への搭載、または親のスマホでの買い物前後の補助利用を検討します。

## 運用イメージ（3パターン）
| | パターン | 位置づけ |
|---|---|---|
| **A** | ピピットセルフアプリ内に子ども向けモードとして統合 | 理想・将来構想 |
| **B** | 店舗の貸出端末に鳩ナビを搭載 | 現実的 |
| **C** | 親のスマホで買い物の前後に補助として使う | 現デモに近い。ただし親のスマホでピピットセルフと鳩ナビを同時に切り替えながら使う運用は非現実的 |

→ 理想形は **A（ピピットセルフ内統合）**、現実解は **B（店舗貸出端末への搭載）**。

## AIの役割（誤解されないために）
- AIは**店内ルートを作らない**。選ばれた商品を**どの順番で回るか（巡回順）を提案するだけ**。
- 実際のナビ・方角・安全・会計は、固定マップ上の売り場情報と方位センサーを使った**アプリ側のローカルロジック**（またはピピットセルフ側）で制御する。
- そのため、AIが不安定な出力をしても、**検証とフォールバック**によって体験が壊れない構成にしている。

## 会計の役割
- 鳩ナビは**会計・登録・精算をしない**。それは**ピピットセルフの役割**。
- 鳩ナビのスキャンは決済処理ではなく、子どもが商品を**「みつけた！」という合図**として使う演出。

## ピピットセルフとは（30秒で説明）
- 平和堂のスマホスキャン型セルフレジ。**東芝テック製**。2022/11 アル・プラザ近江八幡が1号店。
- 流れ：**スマホで商品バーコードを読み取り → 専用精算機で会計**（レジ待ちゼロ）。

## 将来の接続先候補（断定しない・構想段階）
- 東芝テックの店舗プラットフォーム **ELERA** や、パートナー向け **TEC Developers Hub** のAPIが、将来の接続先候補になり得る。
- ただし**現時点では未接続**で、鳩ナビは独立したWebデモとして動作する。具体的な連携方式・API利用は**今後の検討事項**であり、実装済みではない。
- パートナーAPIは登録・審査を前提とするため、実際の接続には平和堂・東芝テックなど当事者との連携が必要になる。

## 今回のデモの実装スタンス（正確に・誤解しないため）
- 入店QRチェックインは実装しない。スタート地点は「入口」固定。
- スキャンは JANコード を読むところまで。会計・POS連携はデモ範囲外。
- 巡回順の提案とクイズ生成に Gemini を使い、失敗時は固定データへフォールバック。
- 途中追加：ナビ画面から「リストにない商品」をスキャン → クイズ → 正解で +1（リスト品はミッション経由、リスト外はこのボタン経由）。
