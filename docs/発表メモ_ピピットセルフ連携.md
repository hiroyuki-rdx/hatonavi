# 発表メモ：ピピットセルフ連携（頭に入れる用）

## 1行で言うと
はとナビは**ピピットセルフを置き換えない**。東芝テック「**ELERA**」プラットフォームの **“ソリューション”として差し込む拡張モード**。

## ピピットセルフとは（30秒で説明）
- 平和堂のスマホスキャン型セルフレジ。**東芝テック製**。2022/11 アル・プラザ近江八幡が1号店、**2024年に200店舗突破**。
- 流れ：**入店QRでチェックイン → 商品バーコードをスキャンしてカゴ登録 → 専用会計機で決済**（レジ待ちゼロ）。HOPマネー対応。

## 仕組み ＝ ELERA の3軸
1. **タッチポイント**：スマホ／カート端末（はとナビUIも同じ枠に同居できる）
2. **ソリューション（マイクロサービス）**：ピピットセルフ本体 ＋ **はとナビ知育ロジックを新サービスとして追加** ←★連携の本体
3. **データサービス**：購買・行動データ、電子レシート連携（→将来のデータ資産）

## 「連携できるの？」への答え
- **設計上できる**。ELERAは“外部ソリューションを差し込む”前提のプラットフォーム。東芝テックは **「TEC Developers Hub」**でパートナー向けAPI（スマホPOS／カート ⇄ 店舗POS）を公開している。
- **条件**：API仕様はパートナー登録＋東芝テックの審査後に公開（学生単独では不可）。**ただし本ハッカソンは平和堂・ソフトバンクが当事者**なので本番連携は現実的。

## 審査員への回答（このまま言える言い回し）
> 「**置き換えではなく、ELERAのソリューションとして差し込みます。** スキャン・決済・POSは既存基盤を流用し、連携境界は TEC Developers Hub のAPI。**デモはこの境界をモック化、本番はパートナーAPIで接続**します。」

→ 「既存POSとの連携アーキ・**腐敗防止層(ACL)**」という技術懸念に直答（**連携境界のAPIが、まさにそのACL**）。

## デモ vs 本番の線引き
- **デモ**：連携境界をモックし“同じ流れ”を再現（スキャン演出→クイズ→決済演出→シール）。
- **本番**：TEC Developers Hub API 経由でピピットセルフ／POS／会計機に接続。
