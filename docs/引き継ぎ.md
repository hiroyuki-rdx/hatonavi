## 2. 絶対に守る方針（DO / DON'T）★最重要★
**DON'T（やってはいけない・表現も使わない）**
- 「**ピピットセルフのファミリーモード**」と言わない（公式統合と誤解される）。
- 「**AIルート生成/計算**」「**最短/最適ルート**」と言わない。
- 鳩ナビに**会計・商品登録・精算を実装しない**（ピピットセルフの役割）。
- AIに**現在地推定・通路ルート生成・方角計算・危険判定・画面遷移・スキャン・精算**をさせない。
- **ナビ中にGeminiを毎回呼ばない**（呼ぶのは開始時の順番提案・商品ごとのクイズ・将来のサマリのみ）。
- UIに「ピピットセルフ」の名前を出さない（連携を聞かれて困るため。将来接続先としてdocsにのみ記載）。
- デモ範囲と将来構想を混ぜない／連携を実装済みと断定しない。

**DO（正しい表現・設計）**
- 位置づけ統一文：「鳩ナビは、ピピットセルフを置き換えるものではなく、将来的にピピットセルフの買い物体験と接続可能な親子向け体験プロトタイプ。現時点は独立Webデモ」。
- 運用3パターン：A=ピピットセルフ内統合(理想)／B=店舗貸出端末(現実的)／C=親スマホで前後補助(デモ)。
- AIは「**巡回順の提案**」と「**食育クイズ生成**」だけ。ナビ・安全・進行はローカル。
- スキャンは「**みつけた！**」のゲームの合図（購入処理ではない）。

## 3. 画面と流れ
`HomeScreen`（スタート＋難易度選択＋「ポイント・シールこうかん」）→ `ShoppingListScreen`（8商品から選ぶ→おつかいの順番）→ `SafetyPledgeScreen`（お約束・音声解放）→ `NavigationScreen`（逐次ミッション＋埋め込み `CompassScreen`、🛒で途中追加）→ `QuizScreen`（スキャン→クイズ→結果）→ `StickerScreen`（おかいけい演出→引換券→図鑑）→ `PointScreen`（ポイント＋スタンプカード＋交換）→ Home。

## 4. ファイル地図（lib/）
- `models.dart`：`ShoppingItem`（id/name/areaId/area/emoji/quiz4択/badge/janCode?）、`sampleItems`（**架空ブランド＋一般名の8品**）、`bonusItem`（途中追加用）、`StoreArea`＋`storeAreas`（**25売り場・pathIndex・x,y座標**）。`hazardAlerts`。
- `services/gemini_service.dart`：`suggestVisitOrder(items)`→`{order:[id]}`（検証＋null）、`generateQuiz(item,{level})`→`GeneratedQuiz?`（**fact=item.explanation を渡しグラウンディング**・検証・null）。同一オリジン `/api/gemini`。
- `services/point_service.dart`：累計P永続化。`pointsPerStamp=2`/`stampsToRedeem=5`/`stickerThreshold=10`、`stampsFor`/`canRedeem`/`redeem`。
- `services/level_service.dart`：3レベル（未就学/小学生/高学年・既定=小学生）、`currentId`。
- `services/compass_service.dart`：方位センサー（絶対方位優先・円環スムージング）、`baselineOffsetDeg`/`storeNorthOffsetDeg`。
- `services/motion_service.dart`：加速度で走行検知（早歩き超え0.5秒で発火）。
- `services/speech_service.dart`：Web音声（`unlock()`＋`speak()`）。
- `screens/`：home/shopping_list/safety_pledge/navigation/compass/quiz/sticker/point。
- `widgets/`：`hatoppy_widget.dart`（マスコット）、`sticker_ticket.dart`（引換券）。
- `api/gemini.js`：Vercel Function。mode `order`/`quiz`。**gemini-2.5-flash**、キーは環境変数 `GEMINI_API_KEY`。quizは fact グラウンディング＋temperature 0.3。

## 5. AI（Gemini）
- **GEMINI_API_KEY は Vercel の環境変数に設定済み**（変更時は再デプロイで反映）。キーはフロントに出さない。
- ハルシネーション対策（多層）：①事実グラウンディング（`item.explanation`を唯一の根拠）②temp0.3＋厳格プロンプト③フロント多段検証（4択/重複なし/index/空）④失敗時は手書き固定クイズへフォールバック＝**誤クイズを表示しない**。

## 6. ビルド/デプロイ/確認
- ローカルビルド：`flutter build web --release`（Windowsはシンボリックリンク警告が出るが web ビルドは成功する）。
- `flutter analyze`：error 2件は `dart:js_util`（compass/motion）の**解析偽陽性**＝無視可。実害なし。
- デプロイ：`main` に push → Vercel 自動。
- 実機確認：スマホのChrome/Safariで `https://hatonavi.vercel.app`。カメラ・センサーは許可。iOSは方位/カメラ権限がタップ起点、サイレントONだと音声は鳴らない。

## 7. 開発の進め方（このプロジェクトの流儀）
- **マルチエージェント**で並列開発（独立ファイル単位）＋**総チェックエージェント（QA）を必ず最後に回す**。不備があれば修正エージェントを呼ぶ。
- **ドキュメントを唯一の正に保つ**（`docs/仕様書.md`）。変更したらdocsも更新。
- フィーチャーはブランチ分離→ビルド→（必要なら/code-review）→push→実機確認。
- 禁止表現 grep（`ファミリーモード|AIルート|最短ルート|最適ルート|planRoute`）＝0件を維持。
