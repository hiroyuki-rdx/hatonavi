# 要件定義書

**プロジェクト名:** はとナビ おつかいクエスト  
**バージョン:** 1.0（デモプロトタイプ）  
**作成日:** 2026-06-20  
**技術スタック:** Dart / Flutter Web → Vercel

---

## 1. プロジェクト概要

子どもが保護者と一緒にスーパーマーケットでおつかいをするとき、  
安全に移動しながら商品を探し、食育クイズに答えてポイントを貯める  
**冒険型おつかい体験アプリ**。

対象ユーザー：小学校低学年の子ども（操作）と保護者（準備・精算）  
対象店舗イメージ：平和堂（琵琶湖・滋賀テーマの食育コンテンツ）

---

## 2. ビジネス要件

| ID | 要件 | 優先度 |
|----|------|--------|
| BR-01 | 子どもが単独で操作でき、おつかいを完遂できること | 必須 |
| BR-02 | 歩きスマホ・走り回りを防止する安全設計を持つこと | 必須 |

## 3. 機能要件

### 3-1. ホーム画面
- 「クエストをはじめる」ボタンで買い物フローを開始する
- 「ポイント交換」ボタンで直接シール交換フローへ遷移する（将来）

### 3-2. 買い物リスト画面（親ターン）
- 商品リストをタップしてチェック・アンチェックできる
- 検索ボックスは UI として存在するが機能しない（ダミー）
- 1件以上選択しないと次へ進めない
- 「AIルートを計算する」ボタンで擬似ローディングを経て遷移する

### 3-3. お約束画面（引き渡し）
- 「はしらない」「まえをみる」の約束をタップして確認する
- 確認後にスマホを子どもへ渡すよう案内する
- **【追加実装】** この画面を新規作成する必要がある

### 3-4. 方位磁針画面（子どもターン・移動中）
- デバイスの方位センサー（DeviceOrientationEvent）を使い、コンパスを表示する
- 画面下部に「ゆっくりあるこう」を常時表示する
- 移動中は画面ロック（歩きスマホ防止）を演出する
- **【追加実装】** 現状は `navigation_screen.dart` 内のロック演出のみ。独立した compass 画面を追加する

### 3-5. 危険エリア警告
- 事前登録された危険箇所に近づくと警告ポップアップを表示する
- ポップアップ表示中は画面を赤くフラッシュする
- 現在はデモ用の固定文言。将来はマップ連動で自動判定する

### 3-6. 棚エリア到着・ミッション表示
- 次に向かう棚エリアを 1 つずつ表示する
- 「棚エリアに到着！」ボタンで次のフェーズへ進む
- 進捗バー（何件目 / 全件数）を表示する

### 3-7. JAN コードスキャン画面
- 「商品をスキャンする」ボタンで擬似スキャン演出を行う（現状）
- **【追加実装★】** `flutter_barcode_scanner` 等のパッケージで実際の JAN コードを読み取り、商品 ID と照合する

### 3-8. クイズ画面
- スキャンした商品に対応した 4 択クイズを表示する
- 正解すると 1P 獲得・限定バッジをアンロックする
- はとっぴーキャラクターが問題・解説を話す演出をつける
- **【追加実装★】** Gemini API で商品情報からクイズを自動生成する

### 3-9. 完了・ポイント確認画面
- 全商品を回ったら「おつかい大成功！」を表示する
- 「大人にスマホを返してね」の案内を表示する
- 今回獲得ポイントと累計ポイントを表示する
- 累計ポイントが交換基準を超えたらシール交換ボタンを活性化する

### 3-10. シール引換券表示
- 疑似 QR コードを表示し、サービスカウンターで提示できる画面を表示する

## 4. 非機能要件

| カテゴリ | 要件 |
|---------|------|
| **デザイン** | ミントグリーン・グリーン基調。ボタンは角丸・太め・ポップ |
| **フォント** | 子どもが読みやすいひらがな中心のテキスト |
| **レスポンシブ** | スマートフォン縦向き（375px〜428px 幅）を主対象とする |
| **パフォーマンス** | Vercel CDN 配信により 3G 回線でも 3 秒以内に初期表示する |
| **安全性** | HTTPS 必須（センサー・カメラ API のため）。COOP/COEP ヘッダーを設定する |
| **オフライン** | 商品データ・クイズデータはアプリにバンドルし、ネット不要で動作する |
| **プライバシー** | 子どもの位置情報・カメラ映像をサーバーへ送信しない（将来も） |
| **アクセシビリティ** | 最小タップ領域 48×48 dp。フォントサイズ 14pt 以上 |

---

## 5. 制約事項

- Web（Flutter Web）で動作すること。ネイティブアプリのビルドは対象外（デモ段階）
- Gemini API は無料枠（gemini-2.0-flash）を使用し、API キーをフロントに露出しない
- JAN コードスキャンは Web 環境のカメラ権限が必要なため、HTTPS 必須
- 方位センサーは iOS Safari で `DeviceOrientationEvent.requestPermission()` が必要
