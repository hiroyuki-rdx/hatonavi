# はとナビ おつかいクエスト デプロイ手順

このアプリ（Flutter Web）を Vercel に公開して、スマホの Chrome から使えるようにするまでの手順です。
プログラミング初心者の方でも進められるよう、ひとつずつ説明します。

---

## 0. 前提（事前準備）

### Flutter SDK が入っていること
ターミナル（Windows なら PowerShell）で次のコマンドを実行します。

```bash
flutter doctor
```

`No issues found!` または Web 関連の項目に緑のチェックが付いていれば OK です。
もし Flutter が入っていない場合は、公式サイト（https://docs.flutter.dev/get-started/install ）の手順でインストールしてください。

### Web ビルドが有効になっているか確認
```bash
flutter config --enable-web
flutter devices
```
一覧に `Chrome` や `Web Server` が出ていれば Web ビルドできます。

---

## 1. アプリをビルドする

プロジェクトのフォルダ（`hatopro_01`）の中で、次のコマンドを実行します。

```bash
flutter build web --release
```

成功すると `build/web` というフォルダに、公開用の一式（HTML / JS など）が出力されます。
これが Vercel にアップロードされる中身です。

> メモ：`--release` は本番用の最適化ビルドという意味です。動作確認だけなら `flutter run -d chrome` でローカル起動もできます。

---

## 2. Vercel にデプロイする

デプロイの方法は2つあります。**初心者には (a) GitHub 連携がおすすめ**です。

### (a) GitHub 連携で自動デプロイ（推奨）

一度設定すれば、GitHub に push するたびに自動で本番反映されます。

1. プロジェクトを GitHub のリポジトリに push しておきます。
2. https://vercel.com にアクセスし、GitHub アカウントでログインします。
3. 画面右上の「**Add New...**」→「**Project**」を選びます。
4. 対象の GitHub リポジトリ（`hatopro_01`）を「Import」します。
5. 設定画面で、プロジェクトルートにある `vercel.json` の内容が自動で使われます。
   - Build Command: `flutter build web --release`
   - Output Directory: `build/web`
   - （`vercel.json` に書いてあるので、基本そのままで OK です）
6. 「**Deploy**」ボタンを押します。
7. しばらく待つとビルドが完了し、`https://〇〇.vercel.app` という公開 URL が発行されます。

以降は GitHub に push するだけで、自動的に再デプロイされます。

### (b) CLI で直接デプロイ

GitHub を使わず、自分のパソコンから直接アップロードする方法です。

1. プロジェクトのフォルダ（`hatopro_01`）の中で、次を実行します。

```bash
npx vercel --prod
```

2. 初回は Vercel へのログインや、プロジェクト名などをいくつか聞かれます。基本はそのまま Enter で進めて構いません。
3. 完了すると、本番用の公開 URL（`https://〇〇.vercel.app`）が表示されます。

> メモ：`vercel.json` にビルド設定が書いてあるので、Vercel 側が `flutter build web --release` を実行し、`build/web` を公開してくれます。

---

## 3. スマホの Chrome で開く

1. パソコンに表示された公開 URL（`https://〇〇.vercel.app`）を、スマホに送ります。
   - LINE やメールで自分宛に送る、QR コードにする、などが簡単です。
2. スマホの **Chrome**（iPhone の場合は Safari でも可）でその URL を開きます。
3. アプリが表示されれば成功です。

---

## 4. 注意点（とても大事）

### 方位センサー・カメラは HTTPS 必須
スマホの方位センサー（コンパス）やカメラは、**安全な接続（HTTPS）でしか動きません**。
Vercel の公開 URL は最初から HTTPS（`https://`）なので、この点は**自動的にクリア**されています。安心して使えます。

逆に、ローカルの `http://` や IP アドレス直打ちではセンサーが動かないことがあります。スマホで試すときは必ず Vercel の URL を使ってください。

### iOS Safari ではセンサーの許可ポップアップが出る
iPhone（iOS Safari）では、方位センサーを使うときに
「**動作と方向へのアクセスを許可しますか？**」というポップアップが表示されます。
ここで「**許可**」を押さないとコンパスが動きません。ユーザーには「許可を押してね」と案内してください。

### COOP / COEP ヘッダーについて
`vercel.json` には `Cross-Origin-Opener-Policy` と `Cross-Origin-Embedder-Policy` というヘッダーを設定済みです。
これは将来カメラやバーコードスキャンなどの高度な機能を追加したときに必要になるセキュリティ設定で、今のうちから入れてあります（今は意識しなくて大丈夫です）。
