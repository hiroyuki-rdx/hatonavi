## 一言サマリ（発表の柱）

- **作ったもの**: スーパーでの親子の買い物を知育エンタメ化する **Flutter Web** アプリ（平和堂 × ソフトバンク × 龍谷大学 ハッカソン）。公開URL: `https://hatonavi.vercel.app`
- **技術の目玉**: ① Gemini による「巡回順の提案」と「食育クイズ生成」、② **ハルシネーション対策の多層防御**（誤クイズを画面に出さない）、③ センサー（方位・加速度）と音声を使った**安全配慮**（走行ロック・危険音声）。
- **設計思想**: 「**デモが必ず最後まで動く**」— AI・ネット・センサーが失敗しても、固定データ／北固定／無音などへ自動フォールバックする。

## ★発表時の禁止表現（重要・`docs/引き継ぎ.md §2` 準拠）

スライド・口頭の**どちらでも使わない**こと。

- ❌「ピピットセルフのファミリーモード」（公式統合と誤解される）
- ❌「AIルート生成/計算」「最短/最適ルート」（経路を作っている、という表現）
- ❌ 鳩ナビが会計・精算をする、という表現（会計はピピットセルフの役割）
- ✅ AIは「**巡回順の提案**」と「**食育クイズ生成**」だけ、と言う
- ✅ スキャンは購入処理ではなく「**みつけた！**」のゲームの合図、と言う

> 位置づけ統一文：「鳩ナビは、ピピットセルフを置き換えるものではなく、将来的にピピットセルフの買い物体験と接続可能な親子向け体験プロトタイプ。現時点は独立した Web デモ」。
