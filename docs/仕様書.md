## 0. プロダクトの位置づけ（戦略）

- 平和堂×ソフトバンク×龍谷大学のハッカソン企画。親子の買い物を知育エンタメ化する Flutter Web アプリ。
- **ピピットセルフ（平和堂のスマホスキャン型セルフレジ）の「ファミリーモード」として連携**する設計。代替ではない。
  - スキャン＝ピピットセルフのバーコードスキャン、決済＝専用精算機へのハンドオフ、スタート＝入店QRチェックイン。
- AIルート（巡回順）＝ Gemini、方角＝方位センサー（コンパス）、進行確定＝スキャン、の3点で**測位ハードウェア（BLE/GPS）不要**。

## 1. 画面仕様一覧

### 画面遷移マップ（実装準拠）

```
HomeScreen
  └─→ ShoppingListScreen（買い物リスト）
        └─→ [AIルート計算ダイアログ（擬似1.6秒）]
              └─→ SafetyPledgeScreen（お約束）
                    └─→ NavigationScreen（逐次ミッション・本体）
                          │   ├ 内部に CompassScreen を埋め込み（方位センサーで方角表示）
                          │   ├ 2件目到達時に hazardAlert ダイアログ
                          │   └ 「ここに とうちゃく！」→ QuizScreen
                          └─→ QuizScreen（スキャン演出→クイズ→結果）
                                └─→ 次ミッション or StickerScreen（全完了）
StickerScreen（決済演出→シール引換券→図鑑）
  └─→ PointScreen（獲得P＋累計P・シール交換判定）
        └─→ HomeScreen（pushAndRemoveUntil）
```

## 3. データモデル / サービス仕様

### ShoppingItem（`lib/models.dart`）
`id / name / area / emoji / question / choices(4) / correctIndex / explanation / badgeName / badgeEmoji`
- ★Phase 2 で `janCode`（13桁）を追加し、スキャン照合に使う。

### PointService（`lib/services/point_service.dart`）実装済
```dart
class PointService {
  static const int stickerThreshold = 10;   // 交換に必要な累計P
  static Future<int> loadTotal();            // キー 'total_points'
  static Future<void> add(int points);       // shared_preferences で永続化
}
```

### CompassService（`lib/services/compass_service.dart`）実装済・Web専用
- `Stream<double> get headingStream`：`deviceorientationabsolute`（無ければ`deviceorientation`）を購読。**iOSは `webkitCompassHeading`、Android等は `alpha`→`360-alpha`** で北基準0〜360度に正規化。
- `Future<bool> requestPermission()`：iOS Safari の `DeviceOrientationEvent.requestPermission()` を `dart:js_util` 経由で**タップ起点**で呼ぶ。非iOS/例外時は true。
- 実装は `dart:html` + `dart:js_util`（Web専用）。ネイティブ化時は `motion`/`sensors_plus` 等へ差し替え。

## 6. AIルート＋方位センサーによる逐次ナビ（Q6回答・BLEなし）

1. **巡回順**＝Gemini（店舗エリアマップ＋買い物リストのテキスト→回る順番）。失敗時は固定順にフォールバック（Phase 3）。
2. **現在地**＝連続測位しない。「直前にスキャンしたエリア」を現在地とみなす。スタートは入店QRで確定。
3. **方角**＝静的エリアマップ上の現在→次エリアの相対方位を、方位センサー（コンパス）で実世界の向きに合わせ矢印表示。
4. **逐次提示**＝今向かう1個だけ表示し、完了ごとに次へ。全ルートは見せない（走り出し防止）。
5. **校正**＝マップ座標と磁北の対応（店舗ごとの回転オフセット1個）。デモは北基準仮定/オフセットをハードコードで吸収。
→ BLE/GPS/Wi-Fi測位ゼロ。必要なのは静的エリアマップ1枚と方位センサーのみ。

## 7. Gemini API 連携仕様（Phase 3・将来）

- モデル：**`gemini-2.5-flash`**（`gemini-2.0-flash` は廃止のため不可）。
- 用途：①**ルート策定**（マップ＋リスト→巡回順）②**クイズ生成**（商品名→4択クイズ）。
- キー秘匿：**Vercel Serverless Function（`api/gemini.js`）でプロキシ**し `GEMINI_API_KEY` を環境変数で保持（フロントに出さない）。
- ハルシネーション対策：出力を既知エリア名/テンプレに制約＋事実照合、**失敗時は固定順・既存5問クイズにフォールバック**。
